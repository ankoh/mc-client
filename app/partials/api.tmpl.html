Documentation

GET /cache/status

Returns server version, last update and reachability of the Mendeley API.

GET /cache/entities

Returns the number of entities in the different database tables profile, document, cache_profile, cache_profile_has_cache_document, cache_document, cache_document_has_cache_field and cache_field.

POST /cache/update

Triggers the cache synchronization and returns an entity report. The report contains profiles, documents, unified_profiles, unified_documents, fields and field_associations.

GET /fields

Returns all research fields that have been identified by the crawler. The crawler checks the tags that have been added to a Mendeley publication and merges small spelling variations. Thereby, the original tag is internally trimmed, converted to lowercase and some special characters are removed. Only whitespace and minus are allowed as delimiter between words. (e.g. cyber-physical.,: Systems becomes Cyber-Physical Systems)

GET /profiles

Given the optional parameters profiles and fields, returns all Mendeley Profiles that are associated with either a profile id OR a field id. The crawler only caches those profiles that belong to the configured Mendeley Research Group. If you pass the parameter slim=true, the result will only include name, id and cnt.

GET /documents

Given the optional parameters profiles and fields, returns all documents that are associated with a profile id AND a field id. (e.g. ?profiles=abc,cde&fields=efg,hij returns all documents of the users abc and cde that are associated with either the field efg or the field hij)
